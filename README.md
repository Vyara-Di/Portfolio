# vyara.design — static site

Refactored from a single 3,275-line `index.html` (SPA-style, with project
content built dynamically by JS into a hash-routed overlay) into a plain
multi-page static site: one real HTML file per project, shared CSS/JS
extracted into their own files.

## Structure

```
index.html                     — homepage (hero, work grid, about, contact)
styles.css                     — all shared styles
hero-sculpture.js              — the Three.js hero animation (homepage only)
script.js                      — shared interactions: loader, custom cursor,
                                  magnetic buttons, scroll reveals, nav scroll-spy,
                                  case-study reveals/parallax, lightbox, design
                                  mode, contact clock
projects/
  lg-electronics.html
  my-market.html
  smart-point.html
  reverto-winery.html
  crea-bakery.html
  maynooth-furniture.html
images/                        — NOT included — see "Assets" below
assets/                        — NOT included — see "Assets" below
```

## What changed vs. the original file

- **Real URLs per project.** Each case study is now its own page at
  `/projects/<slug>.html` instead of a `#hash` that opened a JS-built overlay.
  Search engines, screen readers, and no-JS visitors now see the actual
  content in the HTML.
- **Project content moved from JS into HTML.** The old `openProject()`
  function built each project's markup at runtime from a big inline JSON
  blob. That markup is now written directly into each project page. Nothing
  about the visual result changed — the static HTML was generated by
  actually running the original `openProject()` logic once per project and
  capturing its output, so it's a byte-faithful snapshot of what the page
  used to render, just baked in instead of rebuilt every visit.
- **CSS extracted** to `styles.css`, linked from every page.
- **JS split in two:**
  - `hero-sculpture.js` — the WebGL hero piece, only loaded on the homepage.
  - `script.js` — everything else (loader, cursor, reveals, nav, lightbox,
    magnetic buttons, design mode, contact clock). All the *project-building*
    code (JSON parsing, `openProject`, hash routing, popstate handling) was
    removed since it's no longer needed; the *effects/motion/interaction*
    code was kept and, where it depended on elements that only exist on
    project pages (or only on the homepage), guarded so one script works
    everywhere.
- **Work cards** on the homepage now link straight to `projects/<slug>.html`.
  `script.js` still applies the tilted/hand-placed hover styling to them —
  it just no longer builds the cards themselves.
- **Prev/Next case-study links** are real `<a href="other-project.html">`
  links now instead of JS click handlers with a fake overlay transition.
- **"Back to work"** is a real link to `../index.html#work`.

Nothing about the look, motion, or interaction behavior was intentionally
changed. Every generated page was run through the real `script.js` in a
headless DOM to confirm it executes with no errors.

## Editorial redesign (case study pages)

The six `/projects/*.html` pages were redesigned from a standard bento-grid
case-study format into an editorial layout:

- **Opening statement, not a hero banner.** Each page opens with an
  oversized serif title (character-by-character reveal, same mechanism as
  the homepage), a one-line headline, and small metadata — then an
  asymmetrically offset hero image (not a full-bleed generic banner).
- **Five-part narrative arc** per case study: Challenge → Creative Thinking
  → Visual Exploration → Final Experience → Impact. Each is a numbered
  "chapter" with a giant faint ghost numeral, generous whitespace, and a
  narrow offset text column (alternating left/right) rather than a repeated
  grid.
- **Asymmetric image compositions** (`.duo`, `.strip`, `.frame--full`
  cinematic moments, `.proj__parallax` scroll-linked images) built per
  project from that project's own actual asset inventory — no two case
  studies use the same rhythm, but they share one visual grammar.
- **Type stays single-family**, per `tokens.html` — Urbanist, weights
  200–900, no second typeface. Voice and contrast come from weight +
  italic (900 for display titles, matching `tokens.html`'s `.t-title`
  spec for case-study titles; 200 italic for headlines/quotes, matching
  its `.t-quote` spec) rather than from mixing typefaces.
- **"Next case study"** is now a full-width editorial transition (huge
  title, arrow) instead of a small nav bar; the last project ends on a
  quiet closing line instead of just stopping.
- All of this reuses the *existing* reveal/parallax/lightbox JS
  (`.proj__reveal`, `.bento--fly`, `.proj__parallax`, `data-zoomable`) — no
  changes were needed in `script.js` for this redesign, only new CSS and
  new markup composition inside each page's `#projContent`.



1. Push this folder as the repo root (or `docs/` if you prefer, adjusting
   Pages' publish source accordingly).
2. Add the `images/` and `assets/` folders above.
3. In the repo's **Settings → Pages**, set the source to the branch/folder
   you pushed to.
4. Everything is relative except the CV link and the two `og:image` /
   canonical URLs in each page's `<head>`, which assume the domain root is
   `vyara.design`. Update those if you deploy somewhere else first.
